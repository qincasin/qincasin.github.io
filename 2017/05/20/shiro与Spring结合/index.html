shiro与Spring结合

发表于 2017-05-20

笔记总结及大纲

认证思路分析

  1. 获取当前的Subject. 调用SecurityUtils.getSubject();方法
  2. 测试当前的用户是否已经被认证，即是否已经登录。调用Subject 的isAuthenticated() 方法
  3. 若没有被认证，则把用户名和密码封装为 UsernamePasswordToken 对象

    在B/S中如何获取用户名密码

  4. 创建一个表单页面
  5. 把请求提交到Spring MVC的Handler
  6. 获取用户名和密码
  7. 执行登录：调用Subject 的login(AuthenticationToken )方法
  8. 自定义Reaml的方法，从数据库中获取对应的记录，返回给Shiro
    1. 实际上需要继承org.apache.shiro.realm.AuthenticatingRealm 类.
    2. 实现doGetAuthenticationInfo(AuthenticationToken var1) 方法.
  9. 由shiro完成对密码的比对

    密码比对

  • 通过AuthenticatingRealm 的 credentialsMatcher 属性来进行密码的比对

    如何把一个字符串加密为MD5

  • 替换当前Realm 的credentialsMatcher 属性 .直接使用HashedCredentialsMatcher 对象，并设置加密算法即可。

    为什么使用MD5 盐值加密

  • MD5加密不可逆
  • 盐值加密的可以解决密码相同时进行加密后的密码不相同
  • 简单来说单纯使用MD5加密后，如果原密码相同，那么加密后也会出现相同的密码，因此采用盐值可以解决此问题。

    如何做到

  1. doGetAuthenticationInfo 方法返回值创建 SimpleAuthenticationInfo 对象的时候需要使用
    SimpleAuthenticationInfo(principal,credentials,credentialsSalt,realmName);构造器。
  2. 使用ByteSource.Utils.bytes()来计算盐值
  3. 盐值需要唯一：一般使用随机字符串或user id
  4. 使用SimpleHash(hashAlgorithmName,credentials,salt,hashIterations); 来计算盐值加密后的密码的值

    授权怎么实现

  5. 授权需要继承 AuthorizingRealm 类 ，并实现其 doGetAuthorizationInfo 方法
  6. AuthorizingRealm 类继承自 AuthenticatingRealm ，但没有实现AuthenticatingRealm 中的
    doGetAuthenticationInfo 抽象方法，所以认证和授权值需要继承 AuthorizingRealm 就可以了，同时实现它的两个抽象方法

spring 搭建

需要配置的xml

web.xml

  • 配置Spring环境
  • 配置Spring MVC的环境
  • 配置Shiro Filter（这里设置的是所有的都需要Shiro过滤）

spring-servlet.xml

  • 在这里创建需要扫描的包 com.qjx.shiro
  • 启动Spring MVC的注解
  • 设置mvc的handler

applicationContext.xml

  • 配置SecurityManager
    • 配置authenticator属性
    • 配置realms属性（可以多个也可以单个，多个时使用list标签，注意多个时是由顺序的）
    • 配置cookie的有效时长（这个自己随意配置）
  • 配置CacheManager
    • 需要加入ehcache 的jar 包及配置文件.
      • 以可以配置缓存的文件读取
  • 配置ModularRealmAuthenticator认证器
    • 也就是上面的SecurityManager下的authenticator
    • 可以设置或者修改认证策略，让所有的Realm都认证成功后才可以认证成功 默认是一个认证成功即可 AtLeastOneSuccessfulStrategy(返回所有的Realm身份验证成功的认证信息)
  • 配置Realm
    • 直接配置实现了 org.apache.shiro.realm.Realm 接口的bean
    • 设置加密方式以及加密的次数(可以设置多个Realm来进行加密)
  • 配置 LifecycleBeanPostProcessor。可以自动的来调用配置在Spring中的 IoC容器中 shiro bean 的声明周期方法
  • 启用IoC 容器中使用shiro 的注解。但必须在配置了LifecycleBeanPostProcessor 之后才可以使用
  • 配置ShiroFilter
    • id 必须和web.xml 文件中配置的DelegatingFilterProxy 的一致
    • 若不一致，则会破除NoSuchBeanDefinitionException. 因为Shiro 会来IoC 容器中查找和名字对象的filter bean
    • 配置一个bean，该bean 实际上是一个Map，通过实例工厂方法的方式(默认是使用property来配置的，可以自己使用类来配置，当然存储时使用LinkedHashMap来存储键值对)
    • 这里有两个bean 是通过实例化工厂来配置的键值对来进行过滤认证的。
    • 配置shiroService 用于获取Controller里面Session（使用的是Shiro的Session）的，当然也可以前面使用@Component注解来实现

ehcache.xml

设置Shiro的缓存信息使用的，加入此xml后可以在applicationContext.xml中进行做缓存的设置

后台Java代码

handlers包下

ShiroHandler

Controller层，处理登录页面后的跳转，存储Session（可以用于ShiroService里面用Shiro的Session获取等），登录，认证信息，记住我功能判断

  • Annotation形式的Shiro实现
  • 前台登录到后台处理代码
    • 获取当前的Subject
    • 把用户名和密码封装为UsernamePasswordToken 对象
    • remeberme
      • 可以通过前台checkbox传值过来与否来判断是否需要设置记住我这个功能
      • 简单说就是 当前台没有勾选，即后台没有收到值那么就不进行设置记住我这个功能，如果前台勾选了，那么有值传过来了，则设置记住我这个功能
    • 执行登录
    • 跳转到list.jsp

realm包下

ShiroRealm.java

用户登录后进行的MD5加密比对，以及加密盐值。单数据库加密，存储为Mysql数据库时使用MD5加密

  • doGetAuthenticationInfo
  • 把AuthenticationToken 转换为 UsernamePasswordToken
  • 从UsernamePasswordToken 中来获取username
  • 若用户不存在，则可以抛出UnknownAccountException 异常
  • 根据用户信息的情况，决定是否需要抛出其他的AuthenticationException 异常
  • 根据用户情况，来构建AuthenticationInfo 对象并返回,通常使用
    • 以下信息是通过数据库获取的
    • principal:认证的信息，可以是username，也可以是数据表对应的用户的实体类对象。
    • credentials:密码（从数据库中获取的）
    • realmName:当前realm 对象的name，调用父类的getName()方法即可
    • 盐值
  • 授权会被shiro回调的方法doGetAuthorizationInfo
    • 从PrincipalCollection 中来获取登录用户的信息
    • 利用登录用户的信息来获取当前用户的角色或权限（可能需要查询数据库）(如果登录信息里面已经有权限信息则不需要查询数据库)
    • 创建 SimpleAuthorizationInfo 对象，并设置其roles 属性
      • 存放set集合
    • 返回 SimpleAuthorizationInfo 对象

SecondRealm.java

多Realm界面，进行SHA1加密的。可以理解为数据在Oracle里面的SHA1加密，而在MySQL里面进行的是MD5加密

  • doGetAuthenticationInfo
  • 把AuthenticationToken 转换为 UsernamePasswordToken
  • 从UsernamePasswordToken 中来获取username
  • 若用户不存在，则可以抛出UnknownAccountException 异常
  • 根据用户信息的情况，决定是否需要抛出其他的AuthenticationException 异常
  • 根据用户情况，来构建AuthenticationInfo 对象并返回,通常使用
    • 以下信息是通过数据库获取的
    • principal:认证的信息，可以是username，也可以是数据表对应的用户的实体类对象。
    • credentials:密码（从数据库中获取的）
    • realmName:当前realm 对象的name，调用父类的getName()方法即可
    • 盐值
